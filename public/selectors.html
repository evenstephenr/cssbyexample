CSS by Example: selectors

CSS selectors are codified patterns you can use to refer to elements in a document.

We commonly use selectors to refer to - or 'target' - HTML elements by their tag name, or by their class or id attribute.

Example below.

/* tag selectors */ html, body {   /* styles to be attributed to the full HTML document */ } p {   /* styles impacting all p tags in the document */ } /* class selector */ .highlight {   /* will target all elements with the attribute class="highlight" */ } /* id selector */ #some-id {   /* will only target an element with the attribute id="some-id" */ } /* compound selectors */ p.highlight {   /* will only target p class="highlight" elements */ } p.highlight.combo {   /* will only target p class="highlight combo" elements */ } p.highlight a {   /* will target all a tags inside a p tag*/ } p a span#highlight {   /* p -> a -> span */ }

Some key concepts to understand about selectors, shown in the previous example:

  • Selectors are declared independently, and are not nested
  • Selectors are ignored if they are targeting elements that do not exist
  • Selectors are read left-to-right
  • Compound selectors must fully match the targeted elements to take effect

Selectors take effect in the order they are listed in an HTML document. This is referred to as cascading behavior. More on this later.

Also later we will cover more complex selectors - such as combinators, attribute selectors, and pseudo-selectors - that we can use to get really granular with our targeting.
